Processing to Processing.js Graphic and Sound Files

Recently I made a game using processing and processing.js, you can read the post mortem about it here. It was my first time working with processing.js and using sound and graphic files in my game caused me some trouble. In case anyone else has similar problems, here is a breakdown of how I got my sound effects and custom pixel art to work with processing.js:

  • The first thing to know is that there is no data folder in processing.js. Instead all custom files should go in the same folder as the processing sketch.
  • Graphics need to be preloaded into the sketch, otherwise they won’t show up. The code to do this is: 
    /* @pjs preload="example1.png,example2.png,example3.png"; */
    For more exact information, see the documentation page on the processing.js website.
  • If you’re using sprite sheets or separating images from a single preloaded image, the .get() function with a PImage array will not work for separating out the images. I separated each graphic out manually because I was a little short on time.
  • For audio, I used minim.js by Daniel Hodgin. Minim.js is a basic version of the minim library translated into javascript code. It worked very well for the sound I wanted to play in my game. To use minim.js, save the file as whatever you like then load it into your index.html file, so that it looks something like this:
    <!DOCTYPE html> <html> <head> <title>Your title here</title> <script src="WhatYouCalledYourProcessingJSFile.js"></script> <script src="WhatYouCalledYourMinimJSFile.js"></script> </head> <body> <canvas data-processing-sources="SketchName.pde"></canvas> </body> </html>
  • If you’re using minim.js for sound, you do not have to preload the sound files, unlike the graphics files. Just use the minim library like you usually would in your code and include .mp3 and .ogg versions of your sound files in the same folder as your processing sketch.
